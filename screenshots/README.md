# Screenshots

Product screenshots, grouped by area under one subfolder per area (e.g. `frontend/`, `admin-dashboard/`).

## Naming convention

`<feature>-<short-description>.png` (e.g. `shipments-list-view.png`).

## Before committing

Avoid committing screenshots that contain real credentials, tokens, or personal data — use a demo/test account when capturing these.

## Areas

- [frontend/](frontend/) — end-user facing UI
- [admin-dashboard/](admin-dashboard/) — admin/back-office UI
- [demo/](demo/) — screenshots captured during evaluation/demo runs
